# EchoBlog 技术方案文档

## 一、项目概述

EchoBlog 是一款简洁优雅的个人博客系统，后端基于 Go + Gin 框架构建，前端采用原生 JavaScript（无框架依赖），数据存储使用 MySQL + Redis，适合个人博主快速搭建独立的博客平台。

**项目定位**：轻量级自托管博客系统，强调简洁部署和独立数据掌控。

## 二、技术架构

### 2.1 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Go 1.26 / Gin | 高性能 HTTP 框架 |
| ORM | GORM | 自动迁移，链式查询 |

## 五、模块设计

### 5.1 用户认证模块

- **注册**：`POST /api/user/register`，用户名唯一性校验，邮箱格式校验，密码 bcrypt 加密存储
- **登录**：`POST /api/user/login`，用户名+密码验证，成功返回 JWT Token（存储至 Redis 加速验证）
- **获取当前用户**：`GET /api/user/me`，JWT 中间件解析，返回用户基本信息
- **资料更新**：`PUT /api/user/profile`，支持更新昵称（最长 20 字符）、简介（最长 200 字符）

### 5.2 文章模块

- **列表**：`GET /api/posts`，支持分页（page / page_size）和分类筛选（category_id）
- **详情**：`GET /api/posts/:id`，返回文章完整内容、作者信息、分类和标签
- **发布**：`POST /api/posts`，JWT 认证，写入 MySQL，返回文章 ID
- **编辑**：`PUT /api/posts/:id`，JWT 认证，仅文章作者可编辑
- **删除**：`DELETE /api/posts/:id`，JWT 认证，软删除，仅作者可操作
- **我的文章**：`GET /api/my/posts`，JWT 认证，返回当前用户的文章列表

### 5.3 分类模块

- **列表**：`GET /api/categories`，返回所有分类（ID、名称、描述）

## 七、安全设计

| 安全措施 | 实现方式 |
|------|------|
| 密码存储 | bcrypt 加密，不可逆 |
| 身份验证 | JWT Token，存储于 Redis |
| 写操作保护 | 所有写接口（POST/PUT/DELETE）均需 JWT 认证 |
| 权限控制 | 文章编辑/删除仅作者本人可操作（user_id 对比） |
| 参数校验 | Gin binding 标签自动校验必填、邮箱格式等 |
| SQL 注入防护 | GORM 参数化查询，完全避免 |
| 软删除 | 数据库 DeletedAt 字段，防止数据误删 |

## 八、部署架构

### 8.1 部署拓扑

```
                     用户浏览器
                          │
                          ▼ HTTPS
                   ┌──────────────┐
                   │  阿里云 ECS  │
                   │  (Nginx 可选) │
                   │              │
                   │  EchoBlog    │
                   │  (二进制进程) │
                   └──────────────┘
                          │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
         MySQL         Redis         本地文件
         数据库         缓存      (uploads/ 封面/头像)
```

### 8.2 部署步骤

```bash
# 1. 服务器安装 Go（Linux）
wget https://go.dev/dl/go1.26.1.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.26.1.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# 2. 安装 MySQL
sudo apt install mysql-server

# 3. 创建数据库
mysql -u root -p
CREATE DATABASE IF NOT EXISTS echoblog DEFAULT CHARACTER SET utf8mb4;

# 4. 安装 Redis
sudo apt install redis-server

# 5. 克隆代码并编译
git clone https://github.com/weilele0/EchoBlog.git
cd EchoBlog
go build -o echoblog main.go

# 6. 配置 config/config.yaml（参考上方配置章节）
# 7. 创建 uploads 目录
mkdir -p uploads

# 8. 使用 systemd 管理进程（可选）
sudo vim /etc/systemd/system/echoblog.service
# [Unit] Description=EchoBlog [Service] ExecStart=/path/to/echoblog
sudo systemctl daemon-reload && sudo systemctl enable echoblog && sudo systemctl start echoblog
```

> ⚠️ 注意：`config/config.yaml` 包含数据库密码，**不要提交到 Git**，已通过 `.gitignore` 过滤。

### 8.3 环境变量（可选）

可通过环境变量覆盖配置文件中的值，Viper 支持从 `ENV`、`flags` 或 `.env` 文件读取。

## 九、项目特点

1. **零前端依赖**：纯原生 JS，无 Vue/React 等框架，降低学习成本和构建复杂度
2. **单文件部署**：后端编译为单一二进制，部署仅需上传一个文件 + `web/` 目录
3. **自动迁移**：GORM 首次启动自动创建/更新数据库表结构，无需手动建表
4. **软删除**：文章删除采用软删除机制，数据可恢复
5. **JWT + Redis 双保险**：Token 签名验证 + Redis 黑名单机制，支持强制登出
6. **SPA 体验**：页面无刷新切换，路由在前端处理，用户体验流畅
